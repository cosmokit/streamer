# Форматы файлов прокси

## Поддерживаемые форматы

### 1. TXT (текстовый файл)
Один прокси на строку. Поддерживаемые форматы строк:

```
# Формат 1: IP:PORT (без авторизации)
203.142.75.91:8080

# Формат 2: IP:PORT:USERNAME:PASSWORD
185.223.95.120:3128:user2proxy:SecurePass456

# Формат 3: USERNAME:PASSWORD@IP:PORT
premium_user3:MyP@ss789@91.203.15.44:8888
```

**Пример файла:** `proxies-example.txt`

### 2. CSV (comma-separated values)
Первая строка - заголовок (ip,port,username,password)

```csv
ip,port,username,password
203.142.75.91,8080,proxyuser1,pass123secure
185.223.95.120,3128,user2proxy,SecurePass456
91.203.15.44,8888,premium_user3,MyP@ss789
```

**Пример файла:** `proxies-example.csv`

### 3. JSON
Массив объектов с полями: ip, port, username, password

```json
[
  {
    "ip": "203.142.75.91",
    "port": 8080,
    "username": "proxyuser1",
    "password": "pass123secure"
  },
  {
    "ip": "185.223.95.120",
    "port": 3128,
    "username": "user2proxy",
    "password": "SecurePass456"
  }
]
```

**Пример файла:** `proxies-example.json`

## Логика работы

1. **Загрузка** - Пользователь загружает файл → прокси получают статус `pending`
2. **Активация админом** - Админ видит все прокси и может активировать → статус `active`
3. **Отображение юзеру** - Пользователь видит только прокси со статусом `active`, `online`, `offline`
4. **Уникальность** - Прокси проверяются по IP адресу, дубликаты пропускаются

## Статусы прокси

- `pending` - Ожидает активации админом (не виден юзеру)
- `active` - Активирован админом (виден юзеру)
- `online` - Онлайн (проверка работоспособности)
- `offline` - Офлайн (недоступен)

## Ограничения

- Максимум **1000 прокси** на пользователя
- Размер файла: до **2 MB**
- Поддерживаемые расширения: `.txt`, `.csv`, `.json`
