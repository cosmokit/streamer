### 4. API Endpoints (через curl)

```bash
# Логин
curl -c /tmp/test_cookies.txt -X POST "http://localhost:8080/api/login" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"email":"user1@streamer.local","password":"user12345"}'

# Получить прогресс
curl -b /tmp/test_cookies.txt "http://localhost:8080/api/progress" \
  -H "Accept: application/json"

# Начать шаг 2
curl -b /tmp/test_cookies.txt -X POST "http://localhost:8080/api/progress/2/start" \
  -H "Accept: application/json"

# Отметить шаг 2 как выполненный
curl -b /tmp/test_cookies.txt -X POST "http://localhost:8080/api/progress/2/complete" \
  -H "Accept: application/json"
```

**Ожидаемые результаты:**
- [ ] `/api/progress` возвращает JSON с `data` и `summary`
- [ ] `/api/progress/{step}/start` меняет статус на `in_progress`
- [ ] `/api/progress/{step}/complete` меняет статус на `awaiting_confirmation`

## Полный Workflow

### Шаг 1: Пользователь начинает обучение
1. User 1 → `http://localhost:8080/dashboard/progress`
2. Нажимает "Начать обучение" на Шаге 1
3. Открывается YouTube в новой вкладке
4. Статус шага → `in_progress`

### Шаг 2: Пользователь отмечает как выполненное
1. Изучает материал
2. Нажимает "Отметить как выполненное"
3. Статус шага → `awaiting_confirmation`
4. Показывается "Ожидает проверки администратора"

### Шаг 3: Администратор подтверждает
1. Admin → `http://localhost:8080/admin/users/2/edit`
2. Нажимает "Просмотр прогресса"
3. Видит шаг со статусом "Ожидает подтверждения" (жёлтый бейдж)
4. Нажимает "Подтвердить"
5. Статус шага → `completed`

### Шаг 4: Пользователь видит обновление
1. User 1 обновляет `/dashboard/progress`
2. Шаг отмечен как "ВЫПОЛНЕНО" (зелёный бейдж)
3. Показывается "Подтверждено" с галочкой
4. Шкала прогресса обновилась: 25% (1 из 4)

### Шаг 5: Полное завершение
1. User 1 завершает все 4 шага
2. Администратор подтверждает все
3. Шкала прогресса → 100%
4. Раздел "Монетизация" разблокируется
