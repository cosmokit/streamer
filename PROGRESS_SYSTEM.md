# Система Прогресса Обучения

## Архитектура

### Таблицы БД

**`learning_steps`** - Справочник шагов обучения (создаются админом):
- `id` - ID шага
- `title` - Заголовок
- `description` - Описание
- `external_url` - Ссылка на обучающий материал
- `order` - Порядок отображения
- `is_active` - Активность (boolean)

**`user_learning_progress`** - Прогресс пользователей:
- `id` - ID записи
- `user_id` - ID пользователя
- `learning_step_id` - ID шага
- `status` - Статус (`new`, `in_progress`, `awaiting_confirmation`, `completed`)
- `started_at` - Дата начала
- `completed_at` - Дата завершения

---

## Статусы Шагов

1. **`new`** - Новый (начальный статус, создаётся автоматически)
2. **`in_progress`** - В процессе (после нажатия "Начать обучение")
3. **`awaiting_confirmation`** - Ожидает подтверждения (после "Отметить как выполненное")
4. **`completed`** - Выполнено (после подтверждения администратором)

---

## API Endpoints

### **GET /api/progress**
Получить все шаги с прогрессом пользователя.

**Response:**
```json
{
  "data": [
    {
      "id": 1,
      "title": "Шаг 1: Регистрация и настройка профиля",
      "description": "Создайте аккаунт и заполните основную информацию о себе...",
      "external_url": "https://www.youtube.com/watch?v=...",
      "order": 1,
      "status": "new",
      "started_at": null,
      "completed_at": null
    }
  ],
  "summary": {
    "total": 4,
    "completed": 0,
    "progress_percentage": 0
  }
}
```

### **POST /api/progress/{step}/start**
Начать обучение по шагу.
- Статус меняется на `in_progress`
- Устанавливается `started_at`

### **POST /api/progress/{step}/complete**
Отметить шаг как выполненный.
- Статус меняется на `awaiting_confirmation`
- Шаг отправляется администратору на проверку

## Админ Панель

### **Управление Шагами**
`/admin/learning-steps`

CRUD операции:
- Создать шаг (title, description, external_url, order, is_active)
- Редактировать шаг
- Удалить шаг
- Изменить порядок

### **Просмотр Прогресса Пользователя**
`/admin/users/{user}/progress`

Администратор видит:
- Все шаги пользователя
- Статусы каждого шага
- Даты начала и завершения
- Кнопка **"Подтвердить"** для шагов со статусом `awaiting_confirmation`

При нажатии "Подтвердить":
- Статус меняется на `completed`
- Устанавливается `completed_at`
- Обновляется шкала прогресса пользователя

## Frontend

### **Страница "Мой прогресс"**
`/dashboard/progress`

**Отображение:**
1. Фазы: "Станьте частью нашей команды" (фаза 1) → "Монетизация" (фаза 2)
2. Шкала прогресса (заполняется по количеству выполненных шагов)
3. Карточки шагов с кнопками

**Логика кнопок:**
- **Статус `new`**: Показывается "Начать обучение"
- **Статус `in_progress`**: Показываются обе кнопки ("Начать обучение" + "Отметить как выполненное")
- **Статус `awaiting_confirmation`**: Показывается текст "Ожидает проверки администратора"
- **Статус `completed`**: Показывается "Подтверждено" с галочкой

**Монетизация:**
- Раздел заблокирован (opacity: 0.5) пока прогресс < 100%
- Разблокируется автоматически при `progress_percentage === 100`

---

## Workflow

### Пользователь:
1. Заходит в "Мой прогресс"
2. Нажимает "Начать обучение" → открывается external_url в новой вкладке, статус → `in_progress`
3. После изучения нажимает "Отметить как выполненное" → статус → `awaiting_confirmation`
4. Ожидает подтверждения администратора

### Администратор:
1. Заходит в профиль пользователя (`/admin/users/{id}/edit`)
2. Нажимает "Просмотр прогресса"
3. Видит шаги со статусом `awaiting_confirmation`
4. Нажимает "Подтвердить" → статус → `completed`
5. Шкала прогресса пользователя обновляется

## Демо Данные

4 шага обучения создаются через `LearningStepSeeder`:
1. Регистрация и настройка профиля
2. Изучение основ стриминга
3. Загрузка и настройка прокси
4. Первый тестовый стрим

## Примечания

- Пользователь не может самостоятельно отметить шаг как `completed` - требуется подтверждение админа
- Шкала прогресса рассчитывается автоматически: `(completed_count / total_count) * 100`
- Монетизация разблокируется только при 100% прогрессе (все шаги `completed`)
- Шаги отображаются в порядке, указанном в поле `order`
